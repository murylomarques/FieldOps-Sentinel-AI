# Architecture

FIELDOPS Sentinel AI follows a modular monorepo design with clear separation between UI, API, ML pipeline, and operational scripts.

## Runtime Components
- **Frontend (Next.js)**: command center, order views, recommendation queue, monitoring, replay
- **Backend (FastAPI)**: auth, domain APIs, orchestration, policy controls
- **PostgreSQL**: transactional store for orders, recommendations, risks, events, and audit logs
- **ML Artifacts**: local model files generated by training scripts

## Agent Pipeline
1. **IntakeAgent** validates and normalizes payloads
2. **RiskScoringAgent** returns probability scores for 4 risk dimensions
3. **ScenarioGenerationAgent** creates feasible intervention scenarios
4. **OptimizationAgent** chooses the best scenario (CP-SAT or fallback heuristic)
5. **PolicyGuardAgent** enforces hard constraints and approval requirements
6. **ExplainabilityAgent** generates deterministic summaries
7. **Executive report and replay services** aggregate and reconstruct outcomes

## Data and Audit Principles
- every recommendation has a `decision_id`
- every request gets a `request_id`
- critical actions generate `audit_logs`
- timeline reconstruction uses `order_events`

## Design Notes
- Local-first implementation
- Deterministic behavior preferred for reproducibility
- No external paid APIs required for demo operation
